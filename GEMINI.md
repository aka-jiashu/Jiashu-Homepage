# Jiashu Profile & Blog

## Project Overview
This project is a personal home site for "嘉树 Jiashu", combining a React-based landing page with a static documentation blog generated by MkDocs. The site focuses on psychology, nature/sports, and technology/AI, centered around the tagline: "Thinking via Code & Nature".

## Tech Stack
- **Frontend:** React 19, TypeScript, Vite 6
- **Styling:** Tailwind CSS 4, Lucide React (icons)
- **Documentation:** MkDocs with Material theme

## Setup & Installation

### Prerequisites
- Node.js (v18+)
- Python 3 (for MkDocs)

### Installation
1.  **Install Node dependencies:**
    ```bash
    npm install
    ```
2.  **Setup Python Virtual Environment for MkDocs:**
    ```bash
    python3 -m venv venv
    source venv/bin/activate
    pip install mkdocs-material
    ```

## Development & Deployment

### Utility Scripts
- `./start_dev.sh`: Starts both the Vite dev server and MkDocs serve in parallel.
- `./deploy.sh`: Builds both the React app and MkDocs blog, then prepares files for deployment.

### Landing Page (React)
- **Start Dev Server:** `npm run dev`
- **Build for Production:** `npm run build` (Outputs to `dist/`)

### Documentation (MkDocs)
- **Serve Docs Locally:** `mkdocs serve` (Defaults to `localhost:8000`)
- **Build Docs:** `mkdocs build` (Outputs to `dist/blog`)

## Architecture & Structure

- **Landing Page (`/`):**
    - Main entry point: `index.html` -> `index.tsx` -> `App.tsx`.
    - Components: `Hero`, `FeatureCard`, `Navigation`, `Footer`, and `FollowModal`.
    - `FollowModal` provides links to Twitter/X, GitHub, and shows the WeChat Public Account ("博约精一") QR code.
    - Images: Dynamically loaded from `imgs/` using `import.meta.glob`. Backgrounds are randomized per section on load.

- **Blog (`/blog/`):**
    - Powered by MkDocs Material theme.
    - Configuration: `mkdocs.yml`.
    - Builds into `dist/blog` to coexist with the React app.

## Key Files & Directories

- `App.tsx`: Main React application logic and UI.
- `docs/`: Markdown source files for the blog.
- `imgs/`: Static image assets (hero, psychology, nature, tech).
- `nginx.conf`: Configuration for serving the site via Nginx.
- `dist/`: Unified production build output.
